泛型编程：运行程序员在编写代码时使用一些以后才指定的类型，在实例化时作为参数指明这些类型。C++中称之为模板
C++使用关键字 template <typename T> 设置模板。模板类型参数T将作为占位符类型，被替换为参数传入的类型。

<> 中也可以传入非类型参数，如 template <typename T, int size>， 其中size是非类型参数
非类型参数包括：整型、指针引用、浮点类型（c++20） 。 使用时，传入非类型参数的值必须是constexpr值

模板实例化

模板的实例化方式：编译器制作一个副本，将模板参数替换为用户需要的实际数据类型，然后编译该副本。
模板未被使用时，编译器不会对其代码进行完整的语法检查和编译

模板类的实例化：

  • 与普通类不同，为了执行实例化，编译器必须同时看到完整的类模板定义（而不仅仅是声明）和所需的特定模板类型。
  • 因此不能将类定义放在标头中，而将成员函数定义放在 .cpp 文件中。

解决模板类函数未实例化的方法：
1. 将所有模板类代码放在头文件.h中,而不是将模板定义放在.cpp文件中
2. 也可以将模板的.cpp 的内容移动到名为 .inl 内联文件中，在.h文件中包含.inl
3. 添加第三个文件管理模板，并在这个文件中包含所有要实例化的类。

函数模版专用化

允许我们为特定类型或值显式定义模板的不同实现。

  • 当所有模板参数都是专用化时，它称为完全专用化。
  • 当只有某些模板参数是专用化的时，它称为部分专用化。
    • 部分模板专用化只能用于类成员函数，而不能用于模板函数
    • 模板函数必须是完全专用化的

上面例子中的模板专有化
1. 首先，我们需要一个模板参数声明，这样编译器就知道我们正在做一些与模板相关的事情。但是，在这种情况下，我们实际上不需要任何模板参数，因此我们使用一对空的尖括号。
2. 专用化必须与主模板具有相同的签名（除了专用化在主模板使用 T 的任何地方都会替换 double）。由于主模板具有 const T& 类型的参数，因此专用化必须具有 const double& 类型的参数。当主模板使用 引用传参时，专用化不能使用值传参（反之亦然）。

注意，匹配时永远是精确匹配的非模板函数优先于模板函数。
通常，应尽可能避免使用函数模板专用化，而应使用非模板函数。

完全特化不是隐式内联的（部分特化是隐式内联的）。
如果您将完整的专用化放在头文件中，则应将其标记为内联，以便在包含在多个翻译单元中时不会导致 ODR 冲突。

类模版专用化

类模板特化允许我们为特定数据类型（或数据类型，如果有多个模板参数）专门化模板类。
类模板特化被视为完全独立的类，即使它们的实例化方式与模板化类相同。
相当于允许有多个同名但不同实现的类

NOTE：
- 为了使用专用类，编译器必须能够看到非专用类和专用类的完整定义。因此专用类和函数通常在非专用类定义正下方的头文件中定义
- 单独在文件中使用专用类：如果在单个翻译单元中只需要专用化，则可以在该翻译单元的源文件中定义该专业化。由于其他翻译单位将无法看到专业化的定义，因此它们将继续使用非专业化版本。
