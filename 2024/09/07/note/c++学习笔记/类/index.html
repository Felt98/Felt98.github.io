    面向对象程序设计（object-oriented programming）的核心思想是数据抽象、继承和多态。

    • 抽象： 将类与接口的实现进行分离
    • 继承： 允许从现有类派生出新类的方法，新类可以继承现有类的属性和方法，还可以添加新的属性和方法或者覆盖现有的方法。
    • 多态： 实体具体多种形态，分为编译时多态（静态多态）和运行时多态（动态多态），详情见虚函数
      其他概念：
    • 接口：
    • 封装：将数据和操作数据的函数组合在一起，这样可以隐藏对象的内部细节。
    • 数据隐藏
    • 泛型编程：是通过模板（template）机制允许用户编写与类型无关的通用代码
    • RAII（Resource Acquisition Is Initialization）是C++中的一种编程技巧，它的核心思想是：资源的管理（如内存、文件句柄、锁等）应该与对象的生命周期绑定，即对象在创建时获得资源，在销毁时释放资源。
      面向对象与面向过程的区别：

    类

    物体有两个主要组成部分：1） 一些可观察的属性（例如重量、颜色、大小、固体、形状等），以及 2） 它们可以执行或已经执行的一些基于这些属性的行为（例如被打开、损坏其他东西等）。

    在编程中，我们用变量表示属性，用函数表示行为。对应于类中就是使用成员变量和成员函数
    在类中可以按任何顺序定义成员变量和成员函数！
    在成员函数情况下，我们不需要将 类作为参数传递。调用成员函数的对象将隐式传递给成员函数。因此，调用成员函数的对象通常称为隐式对象。

    如果类类型没有任何数据成员，考虑改用命名空间

    权限访问表：
    注意同一个类的对象之间可以互相访问其私有成员，因为私有成员的访问权限是基于类的
    ![[类-1.png|900]]

    • public（公有）： public访问修饰符允许类成员在类的外部被访问。这意味着，如果一个成员被声明为public，那么它可以在类的任何地方被访问，包括类的实例、派生类以及非成员函数。通常，类的公有成员用于定义类的公共接口，允许外部代码与类进行交互。

    • private（私有）： private私有成员仅在类的内部被访问，或同友元的成员函数访问。这意味着，如果一个成员被声明为private，那么它不能在类的外部或派生类中被访问。通常，类的私有成员用于存储类的内部状态和实现细节。这些成员不应该被外部代码直接访问，以保持封装性和数据隐藏。

    • protected（受保护）： protected访问修饰符介于public和private之间。如果一个成员被声明为protected，那么它可以在类的内部以及派生类中被访问，但不能在其他非成员函数中访问。protected成员通常用于存储在派生类中需要访问或修改的数据或实现细节。

    想要访问或者修改私有成员就要设置getter和setter，例如：

    左值：持久存在，能获取地址，如变量、数组元素、解引用指针
    右值：临时存在，不能获取地址，如字面量、运算结果、临时值

    类与结构的差异：

    1. 类将其成员默认为 private，而结构将其成员默认为 public。
    2. 类默认继承方式是private，结构默认是public
      简单没有特别需求的数据集合建议用struct

    构造函数

    聚合初始化（aggregate initialization）：

    • 如果类的类型是聚合aggregate的（成员都是公共的，非私有），可以使用{}来初始化
    • 成员将按照声明顺序初始化

    构造函数是在创建非聚合类类型对象后自动调用的特殊成员函数。
    - 编译器会查看是否有与调用方提供的初始化值（如果有）匹配的可访问构造函数。
    - 如果找到可访问的匹配构造函数，则先分配对象的内存，然后在未初始化的对象上调用构造函数。反之，没有对应的构造函数就会报错
    - 聚合不允许有构造函数 —— 因此，如果向聚合添加构造函数，它将不再是聚合。

    常见的几种构造函数：

    • 默认构造函数（Default constructor）：不带任何参数（或所有参数具有默认值）的构造函数。如果程序员没有为类定义任何构造函数，编译器将自动生成一个默认构造函数。
    • 带参数构造函数（Parameterized constructor）：带有一个或多个参数的构造函数。这种构造函数允许在创建对象时初始化对象的成员。
    • 拷贝构造函数（Copy constructor）：接受相同类类型对象的引用作为参数的构造函数。它用于初始化一个对象为另一个对象的副本。如果程序员没有定义拷贝构造函数，编译器将自动生成一个。
    • 移动构造函数（Move constructor）：接受相同类类型对象的右值引用作为参数的构造函数。它用于在不创建对象副本的情况下，将资源从一个对象移动到另一个对象。这在C++11及更高版本中引入，以优化性能。如果程序员没有定义移动构造函数，编译器可能会自动生成一个。

    NOTE：

    • 构造函数的隐式转换：构造函数也不例外，Foo（int， int） 构造函数将匹配其参数可隐式转换为 int 的任何调用

      隐式类型转换：编译器将在函数调用中执行参数的隐式转换（如果需要），以便匹配参数为不同类型的函数定义

    • 构造函数不能是 const

      • 通常，不能在 const 对象上调用非 const 成员函数。但是，由于构造函数是隐式调用的，因此可以在 const 对象上调用非 const 构造函数。

    • 构造函数不能是虚函数。

      • 因为虚函数需要完整的对象和虚表，构造函数调用时未构造出完整的对象，也没有虚表，是不能够调用虚函数的

    • 单个列表初始化和单参数构造函数的冲突：

      • C++ 初始化的缺点之一：{ 10 } 将匹配列表构造函数（如果存在）或单参数构造函数（如果列表构造函数不存在）。
      • C++的解决方案是列表不为空时，优先使用列表构造函数。如果为空则默认构造函数优先

    构造函数命名规则

    • 构造函数必须与类同名（大小写相同）。对于模板类，此名称不包括模板参数。
    • 构造函数没有返回类型（甚至没有 void）。

    拷贝构造函数

    • 如果您没有为类提供复制构造函数，C++ 将为您创建一个公共隐式拷贝构造函数。默认拷贝构造函数执行的是逐个拷贝类的成员变量（浅拷贝）
    • 可以使用 = default 语法显式请求编译器为我们创建一个默认的复制构造函数
    • 偶尔我们会遇到不希望某个类的对象是可复制的。我们可以通过使用 = delete 语法将复制构造函数标记为已删除来防止这种情况
    • 自己定义拷贝构造函数：如果类包含指向动态分配内存或资源的指针时，需要自定义拷贝构造函数以实现深拷贝。

    • 隐式复制构造函数调用
      一般需要创建临时对象副本来存储传入的参数时，都会隐式调用复制构造函数，例如下面几种情况。
    1. 将一个类对象初始化新对象
    1. 将对象作为参数传递给函数，且为按值传递时，编译器将隐式调用复制构造函数，复制一个临时副本来传递参数
    1. 函数返回对象，且为按值返回时，从一个函数返回一个临时对象，也会隐式调用复制构造函数。

    const类

    初始化 const 类类型对象后，不允许任何修改const对象数据成员
    Const 对象不能调用非 const 成员函数 （非 const 成员没有限制）
    由于 const 成员函数可以在 const 和非 const 对象上调用，因此如果成员函数不修改对象的状态，则应将其设为 const。
    构造函数不能设为 const，因为它们需要初始化对象的成员

    const 成员函数的限制：
    • 不能修改对象的状态（除非通过 mutable 或 const_cast）。
    • 不能调用非 const 成员函数
    • 无法返回对数据成员的非 const 引用
